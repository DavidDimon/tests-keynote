Testes(javascript)Para que serve e como fazer?

Por quê fazer testes?

  • Garantia que o seu código realmente funciona
  • Serve como documentação
  • Facilita o debug de uma função
  • Confiabilidade
  • Sua vida muito mais feliz!

Tipos de testes

  • Testes unitários
  • Testes de integração
  • Testes de componentes
  • Testes de snapshots

Quais libraries posso usar para fazer testes?

Temos varias libraries, mas as principais são: Enzyme e Jest

Como organizar os testes?

Podemos fazer de dois jeitos

  • Criando a pasta __tests__ na raiz do projeto
  • Criando o arquivo com o sufixo .test.[js/jsx/ts/tsx] no mesmo nivel do arquivo testado

Organização dos testes prós(tests)

  • Todos os testes ficam centralizados em um lugar

Organização dos testes contras(tests)

  • Exige que refaça a mesma estrutura do projeto na pasta __tests__
  • Com o passar do tempo vai ficando um pouco difícil achar os testes

Organização dos testes prós(.test.js)

  • O arquivo de teste fica junto com o arquivo que possui as funções testadas
  • Fácil de localizar onde estão os testes

Organização dos testes contras(.test.js)

  • A pasta onde esta o arquivo de função/component, pode ficar um pouco grande
